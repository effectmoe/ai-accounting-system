# Mastra Quick Start Guide - AAM会計システム

## 現在の状況

Mastra Cloudダッシュボードは現在パブリックベータ版のため、エージェント/ワークフローが表示されない問題があります。しかし、**システムは完全に動作しています**。

## 利用可能な3つの方法

### 方法1: ローカルMastraサーバー（推奨）
```bash
# サーバー起動
npm run mastra:start

# 別ターミナルでエージェント実行
./run-mastra-agent.sh
```

**メリット:**
- 最速の応答速度
- 完全なコントロール
- デバッグが容易

### 方法2: Vercel API経由
```bash
# エージェント一覧を確認
curl https://aam-accounting-mastra.vercel.app/api/agents

# ワークフロー一覧を確認
curl https://aam-accounting-mastra.vercel.app/api/workflows

# 特定のエージェントを実行（例：会計エージェント）
curl -X POST https://aam-accounting-mastra.vercel.app/api/execute \
  -H "Content-Type: application/json" \
  -d '{"agent": "accountingAgent", "task": "請求書を処理してください"}'
```

**メリット:**
- どこからでもアクセス可能
- サーバー管理不要
- 自動スケーリング

### 方法3: 直接Node.jsスクリプト実行
```bash
# 会計ワークフロー実行
npm run mastra:workflow

# 日次自動化実行
npm run mastra:daily

# 月次自動化実行
npm run mastra:monthly
```

**メリット:**
- バッチ処理に最適
- cronでの自動実行可能
- カスタマイズが容易

## 12個の利用可能なエージェント

1. **会計エージェント** - 請求書、領収書、見積書の処理
2. **顧客エージェント** - 顧客情報管理とCRM
3. **データベースエージェント** - MongoDB/Supabaseデータ操作
4. **デプロイメントエージェント** - Vercelデプロイ管理
5. **日本税務エージェント** - 消費税、源泉徴収税計算
6. **OCRエージェント** - 画像からテキスト抽出
7. **問題解決エージェント** - エラー分析と解決
8. **製品エージェント** - 在庫管理と商品情報
9. **リファクタリングエージェント** - コード品質改善
10. **UIエージェント** - React/Next.jsコンポーネント生成
11. **構築エージェント** - システムアーキテクチャ設計
12. **Webスクレイパーエージェント** - Webデータ抽出

## 4個のワークフロー

1. **会計ワークフロー** - OCR → 検証 → 税計算 → DB保存
2. **コンプライアンスワークフロー** - 文書収集 → 法令チェック → レポート生成
3. **請求書処理ワークフロー** - OCR → 顧客照合 → 検証 → 保存
4. **デプロイメントワークフロー** - リファクタリング → UI構築 → パッケージング → デプロイ

## トラブルシューティング

### Mastra Cloudに表示されない場合
これはパブリックベータ版の既知の問題です。実際のエージェントは正常に動作しています。

### ローカルサーバーが起動しない場合
```bash
# 依存関係を再インストール
npm install

# ポートを変更して起動
PORT=4112 npm run mastra:start
```

### Vercel APIがタイムアウトする場合
Vercelの無料プランには10秒の実行時間制限があります。長時間のタスクはローカルサーバーを使用してください。

## 推奨ワークフロー

1. **開発/テスト時**: ローカルMastraサーバー使用
2. **本番運用時**: Vercel API使用
3. **バッチ処理**: 直接Node.jsスクリプト実行

## サポート

問題が発生した場合は、以下を確認してください：

1. Node.js v20以上がインストールされているか
2. MongoDBが起動しているか
3. 環境変数が正しく設定されているか（.envファイル）
